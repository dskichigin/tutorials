# План по Spring Boot

1. Spring Boot (http://start.spring.io)
   1. Spring заменяет конфигурированием программирование некоторых задач
   2. Spring Boot - специальный фреймворк для быстрого создания production-grade приложений
   3. Stand-alone приложения
   4. Embeded Tomcat, Jetty, Undertow сервера JAR
   5. Предоставляет опциональный 'starter' POMs для упрощения Maven конфигурирования
   6. Автоматически конфигурирует Spring где это возможно
   7. Предоставляет production-ready features такие как метрики, самодиагностика (health check) и внешняя конфигурация (Actuator)
   8. Никакой кодогенерации и никаких XML-конфигураций
2. Где полезен Spring Boot?
   1. При старте разработки Spring-проекта
   2. Проектов базирующихся на микросервисной архитектуре
   3. В больших проектах
   4. В маленьких проектах, прототипах, Proof-of-Concept, test/mock-приложениях
3. Микросервисы
   1. Каждая бизнес функция это отдельный сервис
   2. Простые коммуникации (REST или JMS)
   3. Децентрализованное управление, deploy, управление данными
   4. Автоматизация инфраструктуры: автотесты, автодеплой, мониторинг
   5. Сервисы просто заменять
   6. Сервисы могут быть написаны на разных языках
   7. Естественное разделение сервисов
   8. Простая горизонтальная масштабируемость
4. Spring Boot 'starter'
   1. Наборы POM-ов
   2. Упрощают создание Spring Boot приложений
   3. Содержат массу настроек по умолчанию
   4. Содержат все необходимые зависимости
   5. Spring Boot Starter-ы опциональны!!
5. Конфигурирование
   1. @Conditional - магия Spring Boot
   2. Автоконфигурация - любой стартер состоит из автоконфигурации
   3. application.properties
   4. @ConfigurationProperties
